Elite.CSS Documentation

Elite.css uses a non-canonical naming scheme of the BEM methodology, where components are organised into blocks, elements and their respective modifiers.

All classes defined by elite.css are namespaced under ed-*, so they don't conflict with your native styles, like layout rules.

Components

  • Button

Button

The button component is a rectangular, clickable component that can be composed of quite a few elements.

The simplest button is a block element defined with ed-button, and two elements: ed-button__inner and ed-button__main.

The inner element envelops the main corpus of the button, and the main element holds the most prominent content. 

<button class="ed-button">
    <div class="ed-button__inner">
        <div class="ed-button__main">Exit</div>
    </div>
</button>

The most common kind of button in Elite Dangerous has an accompanying icon.

Modify the button block with ed-button--has-head and add the ed-button__head element to give the button a head.

Inside the head element, add your icon block and make sure to mix it with the ed-button__icon element so it gets correctly handled by the block. 

<button class="ed-button ed-button--has-head">
    <div class="ed-button__head">
        <div
            class="ed-icon
                   ed-button__icon
                   ed-icon-universal-cartographics"
        ></div>
    </div>
    <div class="ed-button__inner">
        <div class="ed-button__main">
            Universal Cartographics
        </div>
    </div>
</button>

Add ed-button__sub to allow secondary content within the button's body. 

<button class="ed-button ed-button--has-head">
    <div class="ed-button__head">
        <div
            class="ed-icon
                   ed-button__icon
                   ed-icon-commodity"
        ></div>
    </div>
    <div class="ed-button__inner">
        <div class="ed-button__main">
            Commodities Market
        </div>
        <div class="ed-button__sub">Agriculture</div>
    </div>
</button>
