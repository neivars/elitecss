Key Value Pairs

Key value pairs display a two column layout of labels and their values, one per row.

Within a ed-keyvalue block, nest ed-keyvalue__key and ed-keyvalue__value elements to define the key value pairs.

Keys are presented on the left-hand and values are presented on the right-hand column in the sequential order they are nested in. They can be inserted alternating, or grouped as keys or values.

The most semantic elements for this kind of structure are description lists, terms and details (dl, dt, and dd), but generic div tags will work just as well.

Faction State
Expansion
System Influence
My Reputation
Neutral
<dl class="ed-keyvalue">
    <dt class="ed-keyvalue__key">Faction State</dt>
    <dd class="ed-keyvalue__value">Expansion</dd>

    <dt class="ed-keyvalue__key">System Influence</dt>
    <dd class="ed-keyvalue__value">
        <progress class="ed-progress" value="55" max="100"></progress>
    </dd>

    <dt class="ed-keyvalue__key">My Reputation</dt>
    <dd class="ed-keyvalue__value">Neutral</dd>
</dl>

Tight Key Value Pairs

Modify the block with ed-keyvalue--tight to tighten the key column to be only as wide as it needs to be, allowing the rest of the width to be used by the values column.

Superpower
Federation
System Influence
Government
Confederacy
<dl class="ed-keyvalue ed-keyvalue--tight">
    <dt class="ed-keyvalue__key">Superpower</dt>
    <dd class="ed-keyvalue__value">Federation</dd>

    <dt class="ed-keyvalue__key">System Influence</dt>
    <dd class="ed-keyvalue__value">
        <progress class="ed-progress" value="70" max="100"></progress>
    </dd>

    <dt class="ed-keyvalue__key">Government</dt>
    <dd class="ed-keyvalue__value">Confederacy</dd>
</dl>

Centered Key Value Pairs

Modify the block with ed-keyvalue--centered to align the keys to the central divide between them and the values.

Superpower
Federation
System Influence
Government
Confederacy
<dl class="ed-keyvalue ed-keyvalue--centered">
    <dt class="ed-keyvalue__key">Superpower</dt>
    <dd class="ed-keyvalue__value">Federation</dd>

    <dt class="ed-keyvalue__key">System Influence</dt>
    <dd class="ed-keyvalue__value">
        <progress class="ed-progress" value="70" max="100"></progress>
    </dd>

    <dt class="ed-keyvalue__key">Government</dt>
    <dd class="ed-keyvalue__value">Confederacy</dd>
</dl>

When combined with tight key value pairs, the extra left-aligned space can be reclaimed for the values column.

Name
CMDR Nine Oh Three
Ship
Type-9 Heavy
Balance
9,000,300 CR
<dl class="ed-keyvalue ed-keyvalue--centered ed-keyvalue--tight">
    <dt class="ed-keyvalue__key">Name</dt>
    <dd class="ed-keyvalue__value">CMDR Nine Oh Three</dd>

    <dt class="ed-keyvalue__key">Ship</dt>
    <dd class="ed-keyvalue__value">Type-9 Heavy</dd>

    <dt class="ed-keyvalue__key">Balance</dt>
    <dd class="ed-keyvalue__value">9,000,300 CR</dd>
</dl>

Vertically Spread Key Value Pairs

Key value pairs take up as much vertical space as they need, but can be made to spread within a taller containing parent, though the effects are more drastic the bigger the difference in heights is.

Modify the block with ed-keyvalue--stretched space the key value rows vertically.

Superpower
Federation
System Influence
Government
Confederacy
<div style="height: 150px;">
    <dl class="ed-keyvalue ed-keyvalue--stretched">
        <dt class="ed-keyvalue__key">Superpower</dt>
        <dd class="ed-keyvalue__value">Federation</dd>

        <dt class="ed-keyvalue__key">System Influence</dt>
        <dd class="ed-keyvalue__value">
            <progress class="ed-progress" value="70" max="100"></progress>
        </dd>

        <dt class="ed-keyvalue__key">Government</dt>
        <dd class="ed-keyvalue__value">Confederacy</dd>
    </dl>
</div>
